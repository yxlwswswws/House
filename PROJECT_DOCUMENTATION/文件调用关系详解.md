## 三、重置函数的调用链

### 3.1 hRLHeatingSystemResetFcn.m (训练用)

```
【何时被调用】
  • 每个训练 episode 的开始
  • 通过 env.ResetFcn 函数指针
  • 由 train() 或 sim() 自动调用

【输入与输出】
  输入: in (环境状态结构体，通常为空)
  输出: in (修改后的状态)
  
【内部数据流】

  function in = hRLHeatingSystemResetFcn(in)
    
    % 第1步: 读取基础工作区数据
    data = evalin("base", "temperatureData");
    % 数据形状: [N×2]
    % 列1: 时间戳
    % 列2: 室外温度
    
    % 第2步: 获取配置参数
    numDataPoints = size(data, 1);                    % = 1440
    MaxStepsPerEpisode = evalin("base", "maxStepsPerEpisode");  % = 1000
    SampleTime = evalin("base", "sampleTime");        % = 120
    
    % 第3步: 计算所需数据点数
    dataStepSize = ceil(SampleTime/60);               % = 2
    totalDataSteps = dataStepSize * MaxStepsPerEpisode;  % = 2000
    
    % 第4步: 确定可用的起始索引范围
    lastPossibleDataInitialIndex = numDataPoints - totalDataSteps + 1;
    % 计算: 1440 - 2000 + 1 = -559 (负数!)
    
    % 第5步: 随机选择起始索引
    initialIndex = randi(lastPossibleDataInitialIndex);
    
    % 第6步: 提取从起始点到末尾的数据
    inputData = data(initialIndex:end, :);
    
    % 第7步: 重新生成时间戳 (从60秒开始)
    inputData(:,1) = 60:60:60*size(inputData,1);
    
    % 第8步: 写回基础工作区
    assignin("base", "outsideTemperature", inputData);
    
  end

【关键特点】
  ✓ 随机选择: 不同episode用不同数据
  ✓ 目的: 增加训练多样性
  ✓ 影响: Agent 学到的策略更通用
```

### 4.1 rlHouseHeatingSystem.slx 内部架构

```

  ├─ comfortMin (由奖励块读取)
  └─ sampleTime (模型参数)
         ↓ 通过 evalin() 读取
  Simulink 模块

【模型内部流】

  观察输出 [6×1]
    ├─ obs1: T_room (房间温度)
    ├─ obs2: T_out (室外温度)
    ├─ obs3: comfortMax
    ├─ obs4: comfortMin
    ├─ obs5: a_{t-1} (前一动作)
    └─ obs6: 其他
       ↓
  【RL Agent 块】
    ├─ 读取: agent (从基础工作区)
    ├─ 处理: agent.predict(observation)
    ├─ LSTM 层 → 提取时间特征
    ├─ DQN 层 → 计算 Q 值
    ├─ 探索 → ε-贪心选择
    └─ 输出: action {0, 1}
       ↓
  【加热器块】
    ├─ IF action == 0: P_in = 0 W
    └─ IF action == 1: P_in = 5000 W
       ↓
  【热力学模型】 (Simscape)
    ├─ 方程: C·dT/dt = P_in - K·(T - T_out)
    ├─ 输入: P_in (热功率)
    ├─ 输入: T_out (室外温度, 从查表)
    ├─ 积分: 得到 T_room(t)
    └─ 输出: T_room
       ↓
  【奖励计算块】
    ├─ R_comfort = f(T_room, comfortMin, comfortMax)
    ├─ P_switch = f(action, action_prev)
    ├─ Cost = f(power, price)
    └─ 输出: reward
       ↓
  【数据收集】
    ├─ 输出到 MATLAB: experience 对象
    └─ 包含: Observation, Action, Reward, etc.
```

### 4.2 模块间的数据传递

```
【示例：单个时步的数据流】

时刻 t:

  1. 环境输出观察
     ┌──────────────────────────┐
     │ obs = [T_room_t; T_out_t;│
     │        T_max; T_min;     │
     │        a_{t-1}; other]   │
     └──────────┬───────────────┘
                ↓
  2. RL Agent 处理
     ┌──────────────────────────┐
     │ agent.predict(obs)       │
     │ ├─ LSTM 历史处理         │
     │ ├─ Q 值计算              │
     │ ├─ ε-贪心采样            │
     │ └─ 返回 a_t ∈ {0,1}     │
     └──────────┬───────────────┘
                ↓
  3. 执行动作
     ┌──────────────────────────┐
     │ IF a_t == 0:             │
     │   Heater_Power = 0 W     │
     │ ELSE:                    │
     │   Heater_Power = 5000 W  │
     └──────────┬───────────────┘
                ↓
  4. 仿真 120 秒
     ┌──────────────────────────┐
     │ ΔT = (P - K·(T-T_out))/C │
     │ T_{t+1} = T_t + ΔT       │
     └──────────┬───────────────┘
                ↓
  5. 计算奖励
     ┌──────────────────────────┐
     │ IF 18 ≤ T ≤ 23:         │
     │   R_comfort = 0.1        │
     │ ELSE:                    │
     │   R_comfort = -0.1·...   │
     │                          │
     │ IF a_t ≠ a_{t-1}:       │
     │   P_switch = -0.01       │
     │ ELSE:                    │
     │   P_switch = 0           │
     │                          │
     │ Cost = Power · dt/3.6M   │
     │ R_total = R_comfort +    │
     │           P_switch -     │
     │           Cost           │
     └──────────┬───────────────┘
                ↓
  6. 输出到 MATLAB
     ┌──────────────────────────┐
     │ 记录到 experience:       │
     │ ├─ obs_t                 │
     │ ├─ a_t                   │
     │ ├─ r_t                   │
     │ ├─ obs_{t+1}             │
     │ └─ done_t                │
     └──────────────────────────┘
```

## 七、关键参数的生命周期

### 7.1 agent 对象的生命周期

```
【快速演示模式】
  
  1. 创建: 在前面的训练中创建并保存
  2. 序列化: save("HeatControlDQNAgent.mat", "agent")
  3. 存储: agent → HeatControlDQNAgent.mat (磁盘)
  4. 加载: load() → agent 对象 (内存)
  5. 配置: agent.SampleTime = 120
  6. 共享: assignin("base", "agent", agent)
  7. 使用: Simulink RL Agent 块访问
  8. 推理: agent.predict(observation) 返回动作
  9. 保存: 不修改，仅用于推理
  

【训练模式】
  
  1. 创建: rlDQNAgent() 创建全新 Agent
  2. 初始化: 网络随机初始化
  3. 修改: train() 逐步更新网络权重
  4. 配置: agentOpts 设置各项参数
  5. 使用: 多个episode中不断改进
  6. 评估: rlEvaluator 定期评估性能
  7. 收敛: 当奖励≥85时停止
  8. 保存: save() 存储最终版本到 .mat 文件
```

### 7.2 温度数据的生命周期

```
【原始数据】
  temperatureMar21toApr15_2022.mat (磁盘)

【数据变换】

    ├─ temperatureMarch21 (前1440行, 验证)
    ├─ temperatureData (中间部分, 训练)

【在训练期间】
```
